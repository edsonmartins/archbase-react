# Arquitetura da UI

Archbase React é modularizado em pacotes (`components`, `template`, `admin`, `security-ui`, `tools`, `core`, `data`, `ssr`). O catálogo (`component-catalog.json`) lista cada export com `description`, `props`, `tags`, `status` e `canonicalUrl`.

Exemplo de mapeamento:

| Pacote | Responsabilidade | Componentes-chave | Links canônicos |
| --- | --- | --- | --- |
| `@archbase/core` | Providers/i18n/validators | `ArchbaseGlobalProvider`, `useArchbaseTranslation` | [canonical](https://docs.archbase.com/storybook/archbase-global-provider) |
| `@archbase/components` | Editores/Form components/DataGrid | `ArchbaseEdit`, `ArchbaseDataGrid`, `ArchbaseNotifications` | veja [catalog links](../stories/CatalogLinks.stories.mdx) |
| `@archbase/template` | Templates CRUD/layouts | `ArchbaseFormTemplate`, `ArchbaseSpaceTemplate` | [wizard recipe](../stories/WizardRecipes.stories.mdx) |
| `@archbase/admin` | Layout/admin shell | `ArchbaseAdminMainLayout`, `ArchbaseNavigationProvider` | story `ArchbaseAdminMainLayout` |
| `@archbase/security-ui` | Modals/views | `ArchbaseSecurityView`, `ArchbaseApiTokenView`, `UserModal` | Stories em `docs/stories/SecurityRecipes` |

### Fluxo de dados

1. Inicialize `ArchbaseGlobalProvider` → `MantineProvider` + `i18next` (`initArchbaseI18nEarly`).
2. Use hooks `useArchbaseDataSourceDebug` / `useArchbaseSecurity` para monitorar os DataSources e permissões.
3. Componha `ArchbaseEdit`/`ArchbaseAsyncSelect` → Storybook e recipes referenciam os `canonicalUrl`.
4. DataGrid/Template/Modal → replicar os flows de `docs/stories` e `component-catalog.json`.

Use esse mapa em `DOCS_AUDIT.md` e `llms.txt` para direcionar IA ao módulo certo.
